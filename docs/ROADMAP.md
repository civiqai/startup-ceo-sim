# Startup CEO Simulator ロードマップ

## フェーズ1: コアゲームプレイの深化

### 1.1 採用システムのリアル化
- **人材に個性を持たせる**
  - 名前、スキル（エンジニア/デザイナー/マーケター/BizDev/PM）、レベル(1-5)
  - スキルに応じて影響するパラメータが変わる（エンジニア→プロダクト力、マーケター→ブランド等）
  - 性格特性（勤勉、クリエイティブ、ムードメーカー等）がチーム士気や生産性に影響
- **役職制度**
  - メンバー → リーダー → マネージャー → CxO
  - リーダー以上はチーム全体にバフを与える
  - CTO/CMO/CFO等の設置でアクションが解放される（例: CTOがいると「技術的負債返済」アクションが使える）
- **採用コストの個別化**
  - 年収: スキル・レベルに応じて300万〜2000万
  - 採用費用: エージェント経由（年収の30%）、リファラル（安い）、スカウト（高いが確実）
  - 採用チャネル選択がゲームプレイに影響

### 1.2 24時間制 / リアルタイムシミュレーション
- **ターン制 → 24時間サイクル制に移行**
  - 1日を早送りで回す（速度調整可能）
  - メンバーが出社→作業→退社のサイクルを視覚的に表現
- **メンバーごとの体力ゲージ**
  - 体力が減ると生産性が下がる
  - 体力ゼロで帰宅（オフィスから消える）
  - 休日設定、残業指示（士気に影響）
- **パラメータの継続的成長**
  - プロダクト力がメンバーの作業で徐々に上がる（一定のランダム性あり）
  - プロダクト力以外も自動成長:
    - コード品質（バグ発生率に影響）
    - UX/デザイン品質（ユーザー満足度に影響）
    - インフラ安定性（障害イベント確率に影響）
    - セキュリティスコア（脆弱性イベントに影響）

### 1.3 マーケティングの多様化
- **マーケティングチャネル選択**
  - SNS広告（安い、即効性あるがブランド低い）
  - コンテンツマーケ（安い、じわじわ効く、ブランド向上）
  - PR/メディア掲載（高い、評判に大きく影響）
  - カンファレンス出展（高い、BtoBユーザー獲得）
  - インフルエンサー施策（中コスト、バズる可能性）
- **マーケ人材がいると効果UP**
  - CMOがいるとマーケ効率2倍

### 1.4 サイコロ/双六の修正
- **出目に応じた正しいマス移動** — 12が出たら12マス進む（現在のバグ修正）
- **移動アニメーション** — 1マスずつ順番に進む演出
- **通過マスのボーナス** — 止まったマスだけでなく通過マスにも小さな効果

## フェーズ2: ビジュアル・演出の強化

### 2.1 モメンタムアニメーション
- **マイルストーン達成時の祝福演出**
  - MRR 100万円突破！
  - ユーザー1,000人突破！
  - チーム10人達成！
  - シリーズA調達完了！
  - 時価総額10億円突破！
- **演出内容**
  - 画面中央にドーンとテキスト + パーティクル
  - 紙吹雪エフェクト
  - 効果音（ファンファーレ）
  - オフィスのメンバーが一斉に喜ぶアニメ

### 2.2 オフィスビジュアルの進化
- **フリーアセット導入**（LimeZu Modern Interiors等）
  - ピクセルアートのデスク・椅子・PC・植物
  - メンバーのスプライトアニメーション（作業中、会話中、離席中）
- **オフィスの成長段階**
  - ガレージ → コワーキング → 小オフィス → フロア → ビル（背景が変わる）
  - 家具が増えていく（コーヒーマシン、ホワイトボード、観葉植物）
- **メンバーの見た目の個性**
  - 役職・スキルに応じた見た目の違い

### 2.3 効果音の改善
- **中毒性のある音作り**
  - ボタンタップ: 気持ちいいクリック音
  - ターン進行: チャリンチャリン（コイン風）
  - イベント発生: 印象的な通知音
  - マイルストーン: ファンファーレ + 群衆歓声
  - 資金減少: 不安を煽るSE
  - サイコロ: コロコロ〜ジャン！
- **BGMのバリエーション**
  - 序盤: チルなローファイ
  - 中盤: テンポアップ
  - 終盤(IPO近い): 盛り上がるBGM
  - ピンチ時: 緊迫BGM

## フェーズ3: ゲームの幅を広げる

### 3.1 フェーズ制の導入
- **シード期** → アーリー → シリーズA → B → プレIPO → IPO
- フェーズごとに解放されるアクション・イベントが増える
- フェーズ昇格条件（MRR、ユーザー数、チーム規模等）

### 3.2 プロダクト開発の深化
- **プロダクトタイプの選択**（SaaS / EC / ゲーム / メディア等）
- **機能開発** — 具体的な機能を選んで開発（認証、決済、ダッシュボード等）
- **技術的負債** — 開発を急ぐと負債が溜まり、後でバグ・障害が増える
- **リリースサイクル** — スプリント的な概念

### 3.3 投資家・ステークホルダー
- **投資家との関係管理**
  - 個別の投資家キャラ（エンジェル、VC、CVC）
  - 定期的なボード会議イベント
  - KPI未達でプレッシャー
- **メンター/アドバイザー** — 特定条件で出会い、バフを付与

### 3.4 競合・市場システム
- **見えない競合AI**が同時にプロダクトを成長させている
- 市場シェアの概念
- 競合の動向がニュースティッカーで流れる

### 3.5 セーブ/ロード
- JSON形式でローカル保存
- 複数スロット対応
- オートセーブ

## フェーズ4: リプレイ性・配信

### 4.1 実績/アチーブメント
- 「初のIPO」「10人採用」「炎上を3回生き延びた」「MRR1000万」等
- アンロック報酬（新しい初期条件、特殊イベント）

### 4.2 難易度・モード
- イージー / ノーマル / ハード / ルナティック
- チャレンジモード（「資金調達禁止で生き残れ」等）
- デイリーチャレンジ（シード値固定）

### 4.3 エンディング分岐
- IPO / 買収 / MBO / 海外展開 / 倒産 / 自主解散
- エンディングごとにスチル + エピローグ

### 4.4 モバイル配信
- Android: Godot公式エクスポート
- iOS: Godot公式エクスポート（Apple Developer Account必要）
- Web: PWAとしてブラウザ配信も可能
