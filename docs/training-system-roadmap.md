# 育成・訓練システム 実装ロードマップ

## 概要

採用したメンバーを育成する仕組み。個人訓練（山籠もり、勉強会など）とチームイベント（LT大会、カンファレンス開催など）の2軸で構成。育てるほど強くなるが、外に出すほど転職リスクも上がるジレンマを持つ。

## レベルアップ経験値テーブル

| レベル | 必要累計経験値 | 必要差分 |
|---|---|---|
| Lv.1 → Lv.2 | 100 | 100 |
| Lv.2 → Lv.3 | 350 | 250 |
| Lv.3 → Lv.4 | 850 | 500 |
| Lv.4 → Lv.5 | 1850 | 1000 |

- 毎月のパッシブ成長: +10〜15 exp
- 士気70以上: さらに+5
- 同スキルのリーダー以上がいる: さらに+5
- Lv.1→2 はパッシブだけでも7〜10ヶ月で到達
- Lv.4→5 は訓練を積極的に使わないと現実的に届かない

## 転職リスクシステム

### リスク蓄積

- 勉強会参加: +0.05
- 海外カンファレンス: +0.10
- 自社カンファレンス登壇: +0.03
- 毎月の自然減衰: -0.02（時間が経つと忘れる）

### 引き抜きイベント発生条件

以下の全てを満たすとき、月末に判定:
- `turnover_risk > 0.05`
- `team_morale < 80`（居心地が良ければ残る）
- 給与が `BASE_SALARY + skill_level * 120` 未満（相場以下）
- 発生確率 = `turnover_risk * 50%`（リスク0.10なら5%/月）

### 引き抜きイベントの選択肢

> 「○○に他社からオファーが来ています！年収○○万円を提示されているようです…」

- **昇給して引き留める** → 給与+20%、転職リスクをゼロに、士気+5
- **昇進で引き留める** → 昇進条件を満たしていれば即昇進、転職リスクをゼロに
- **引き留めない** → メンバー退職、残りメンバーの士気-10

## 実装ステップ

### Step 0: データモデル拡張

**目的:** 後続ステップの全ての土台となるフィールドを追加する

**変更ファイル:**
- `scripts/team_member.gd`
- `scripts/save_manager.gd`

**やること:**
1. TeamMember に以下のフィールドを追加
   - `experience: int = 0` — 累計経験値
   - `training: String = ""` — 訓練中なら訓練ID（空なら通常勤務）
   - `training_remaining: int = 0` — 訓練の不在残りターン数
   - `turnover_risk: float = 0.0` — 転職リスク (0.0〜1.0)
   - `exp_multiplier: float = 1.0` — 経験値倍率（熊遭遇等の一時バフ用）
2. レベルアップ判定メソッド `add_experience(amount: int) -> bool` を追加
   - 経験値加算 → EXP_TABLE と比較 → 条件を満たせば skill_level+1
   - 戻り値: レベルアップしたかどうか
   - レベルアップ時に `calculate_salary()` も再実行（給与も上がる）
3. `save_manager.gd` のシリアライズ/デシリアライズに新フィールドを追加
   - 既存セーブデータとの互換性を保つ（デフォルト値でフォールバック）

**完了条件:** 新フィールドがセーブ/ロードで正しく保存・復元される

### Step 1: パッシブ成長（毎月の自然経験値）

**目的:** 訓練UIがなくても「メンバーが育っている」実感を出す

**変更ファイル:**
- `scripts/game_state.gd` — `advance_month()` 内
- `scripts/team_manager.gd` — レベルアップシグナル追加
- `scripts/game.gd` — レベルアップ通知表示

**やること:**
1. `advance_month()` 内で全メンバーに自然経験値を付与
   - 基本: +10〜15 exp（ランダム）
   - 士気70以上: +5
   - 同スキルタイプにリーダー以上のメンバーがいる: +5
   - `exp_multiplier` を乗算（通常1.0、バフ時1.5）
2. TeamManager に `signal member_leveled_up(member, old_level, new_level)` を追加
3. game.gd でシグナルを受けてレベルアップ通知を画面に表示
   - 「🎉 ○○がLv.3にレベルアップ！」的なトースト通知
4. 転職リスクの自然減衰処理も `advance_month()` に追加（-0.02/月、下限0）

**完了条件:** 数ターン経過でメンバーのレベルが上がり、通知が表示される

### Step 2: 訓練データ定義

**目的:** 訓練メニューのマスターデータを1ファイルにまとめる

**新規ファイル:**
- `scripts/training_data.gd`

**やること:**
1. 個人訓練6種 + チームイベント3種の定数Dictionaryを定義
   - 各エントリに `name`, `cost`, `absent_turns`, `exp`, `morale`, `turnover_delta`, `special`, `target`（individual/team）, `reputation`, `brand` を持たせる
2. 特殊イベント定義（訓練IDごとの確率テーブル）
3. 訓練の解放条件（フェーズ制との連動）
   - シード期: オンライン講座、社内LT大会のみ
   - アーリー以降: 勉強会、山籠もり、開発合宿、地獄の新人研修、ハッカソン
   - シリーズA以降: 海外カンファレンス、自社カンファレンス開催
4. ヘルパー関数: `get_available_trainings(phase: int) -> Array`

**完了条件:** データだけが定義されている。UI・ロジックはまだない

### Step 3: 訓練選択ポップアップUI

**目的:** プレイヤーが訓練を選んでメンバーを送り出すUIを作る

**新規ファイル:**
- `scripts/training_popup.gd`

**変更ファイル:**
- `scripts/member_detail_popup.gd` — 「訓練に送る」ボタン追加

**やること:**
1. メンバー詳細画面に「訓練に送る」ボタンを追加
   - 訓練中のメンバーには「訓練中: ○○（残り○ターン）」を表示、ボタンは非活性
2. 訓練選択ポップアップを新規作成（コードで UI 構築、既存ポップアップと同パターン）
   - 上部: 個人訓練タブ / チームイベントタブ の切替
   - 個人訓練: 訓練名・コスト・効果・リスクを一覧表示、各行に「送る」ボタン
   - チームイベント: 同上だが「開催する」ボタン
   - 資金不足の訓練はグレーアウト
   - フェーズでアンロックされていない訓練は 🔒 表示
3. 開発合宿の「2人以上必要」制約のバリデーション
4. 自社カンファレンスの「登壇者選択」UI（メンバー一覧から1人選ぶサブ画面）
5. 確認ダイアログ「○○を山籠もり修行に送りますか？（30万円・1ターン不在）」

**完了条件:** ポップアップが開き、訓練を選択してシグナルが発火する

---

### Step 4: 訓練の実行処理（ターン処理への組み込み）

**目的:** 訓練の開始・進行・完了をゲームループに組み込む

**変更ファイル:**
- `scripts/game.gd` — 訓練ポップアップのシグナル接続、訓練開始処理
- `scripts/game_state.gd` — `advance_month()` に訓練進行処理を追加
- `scripts/team_manager.gd` — 訓練中メンバーの出社スキップ
- `scripts/day_cycle.gd` — 訓練中メンバーを生産性から除外

**やること:**
1. 訓練開始処理（ポップアップから発火）
   - `member.training = "mountain"` 等をセット
   - `member.training_remaining = absent_turns`（0なら即完了扱い）
   - `cash -= cost`
   - 不在0の訓練（オンライン講座・勉強会・LT大会・ハッカソン）は即時効果適用
2. `advance_month()` に訓練進行処理を追加
   - `training_remaining > 0` のメンバーは `training_remaining -= 1`
   - `training_remaining == 0` になったら訓練完了処理を実行
   - 完了処理: 経験値付与、士気変動、転職リスク変動、training をクリア
3. `day_cycle.gd` / `team_manager.gd`: 訓練中（`training != "" && training_remaining > 0`）のメンバーは出社させない（生産性ゼロ）
4. ターンログに訓練結果を表示
   - 「🏔️ ○○が山籠もり修行から帰還！経験値+100」

**完了条件:** 訓練に送ったメンバーが不在→帰還→経験値獲得→レベルアップの一連の流れが動く

---

### Step 5: 特殊イベント（訓練完了時のランダム演出）

**目的:** 訓練結果にランダム性を加えて面白くする

**変更ファイル:**
- `scripts/game_state.gd` — 訓練完了処理内
- `scripts/game.gd` — 特殊イベント演出表示

**やること:**
1. 訓練完了時に `training_data.gd` の特殊イベントテーブルを参照して抽選
2. 各特殊イベントの実装:
   - **覚醒（山籠もり）**: skill_level即+1、専用メッセージ「滝に打たれながらコードの真髄を悟った！」
   - **熊遭遇（山籠もり）**: stamina-30、exp_multiplier=1.5を次回訓練までセット、「熊と戦って生き延びた！逆境に強くなった」
   - **脱走（地獄の新人研修）**: メンバーが退職、「○○は夜中に荷物をまとめて逃げ出した…」→ 残りメンバーの士気-5
   - **覚醒（地獄の新人研修）**: personality を "diligent" に変化、「地獄を乗り越えた○○の目は輝いていた」
   - **人脈獲得（海外カンファレンス）**: reputation+5、「○○が海外VCとコネクションを作った！」
   - **チーム結束（開発合宿）**: 参加者全員の士気+15（通常の+10に加算）
   - **アイデア発見（ハッカソン）**: product_power+5、「徹夜のテンションで画期的なアイデアが生まれた！」
   - **メディア掲載（自社カンファレンス）**: brand+3、users+500
3. 特殊イベント発生時は通常の訓練完了通知とは別に、専用の演出付きポップアップを表示

**完了条件:** 各訓練で特殊イベントが確率で発生し、演出とともに効果が適用される

---

### Step 6: 転職リスクシステム

**目的:** 「育てた人材が辞めるかも」というジレンマを生む

**変更ファイル:**
- `scripts/game_state.gd` — `advance_month()` に引き抜き判定追加
- `scripts/event_manager.gd` または新規でイベント追加
- `scripts/member_detail_popup.gd` — 転職リスク表示追加

**やること:**
1. `advance_month()` の月末処理で引き抜き判定を実行
   - 対象: `turnover_risk > 0.05` のメンバー
   - 条件: `team_morale < 80` かつ 給与が相場以下
   - 発生確率: `turnover_risk * 50%`
2. 引き抜きイベント用のポップアップを作成（またはイベントシステムに統合）
   - 「○○に他社から年収○○万円のオファーが！」
   - 選択肢3つ: 昇給で引き留め / 昇進で引き留め / 見送り
   - 昇給: 給与+20%、turnover_risk=0、士気+5
   - 昇進: 即昇進（条件無視）、turnover_risk=0
   - 見送り: メンバー退職、残りメンバーの士気-10
3. メンバー詳細画面に転職リスクのインジケーター表示
   - リスク0: 表示なし
   - リスク0.01〜0.05: 「😊 安定」（グリーン）
   - リスク0.06〜0.10: 「🤔 そわそわ」（イエロー）
   - リスク0.11以上: 「💭 転職サイト見てる…」（レッド）
4. 高レベル（Lv.4-5）のメンバーは引き抜きオファーの給与が高くなる（引き留めコストが高い）

**完了条件:** 勉強会やカンファレンスに送ったメンバーに引き抜きイベントが発生し、対処を迫られる

## フェーズ制との連動

| フェーズ | 解放される訓練 |
|---|---|
| シード期 (Phase 0) | オンライン講座、社内LT大会 |
| アーリー (Phase 1) | 勉強会、山籠もり修行、開発合宿、地獄の新人研修、ハッカソン |
| シリーズA (Phase 2) | 海外カンファレンス、自社カンファレンス開催 |

## セーブデータ互換性

- 新フィールド（experience, training, training_remaining, turnover_risk, exp_multiplier）は全てデフォルト値を持つ
- 旧セーブデータをロードしても問題なく動作する（experience=0、training=""でフォールバック）
- 訓練中のメンバーのセーブ/ロードも正しく復元される

## 依存関係

```
Step 0 ─→ Step 1 ─→ Step 4
  │                    ↑
  └──→ Step 2 ──→ Step 3
                       ↑
                  Step 5（Step 4完了後）
                       ↑
                  Step 6（Step 4完了後、独立して着手可能）
```

Step 0 は全ての前提。Step 2,3 と Step 1 は並行作業可能。Step 5,6 は Step 4 完了後。
